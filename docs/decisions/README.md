# Architecture Decision Records (ADR)

> 設計上の分岐点で「なぜこちらを選んだか」を残す場所。
> 未来の自分（または別のエージェント）が同じ議論を繰り返さないため。

## 書くとき

以下のどれかに当てはまるなら書く：

- 公開 API に影響する意思決定
- 既存の設計原則（`CLAUDE.md` §5）と衝突しそうな判断
- 要件定義書の決定事項（§9）を修正するとき
- 候補が 2 つ以上あり、選んだ理由が非自明なとき
